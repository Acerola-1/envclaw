## 修复的问题

### ✅ -6. 新建数据播报任务页面优化

**问题**:
1. 推送平台是 hermes 频道，不是平台管理的数据平台
2. 可选技能是多选标签，应该改为下拉列表单选
3. 选了平台后看不到平台的技能
4. 数据平台需要多选，支持一次任务访问多个网站

**修复**:
- 推送平台改为下拉选择，只显示已配置的频道，已配置的排在前面
- 修复了频道配置判断逻辑，从 settingsStore 的单独字段读取配置
- 可选技能改为下拉列表，单选，只显示平台自带的技能
- 当平台切换时，自动更新技能列表
- 平台自带技能显示为标签（不可删除），额外技能通过下拉选择
- 数据平台改为多选，支持选择多个平台
- 合并所有选中平台的功能和技能

### ✅ 0. 添加设置入口

**问题**: 工作台页面没有设置入口

**修复**:
- 在顶栏右侧添加通知按钮和设置按钮
- 设置按钮点击后跳转到 `/hermes/settings` 页面
- 设置页面添加返回按钮，可返回到 envclaw 页面

---

### ✅ 0.5. 推送渠道映射为中文

**问题**: 值守任务的推送渠道显示真实代号（如 `wecom:wo3zARCQAAw6LhYEu94vUAijh5gUQWng`），太长遮挡其他列

**修复**:
- 添加 `formatDeliver()` 函数，将代号映射为中文
- `wecom` → `企业微信`
- `wecom:wo3zARCQ` → `企业微信 · wo3zARCQ`（详情页）
- 添加 title 属性，悬停可查看完整代号

---

### ✅ 1. 浅色模式支持

**问题**: 所有 envclaw 组件硬编码暗色主题，无法响应主题切换

**修复**:
- 创建 `envclaw-theme.scss` 统一主题变量文件
- 22 个 Vue 组件从硬编码颜色改为 CSS 变量
- 覆盖所有页面：工作台/值守任务/平台管理/技能库/运行历史

**效果**: 切换浅色/暗色主题时，所有 envclaw 页面正确响应

---

### ✅ 2. 向导面板挤压主页布局

**问题**: 点击"数据播报"后，向导面板用 flex 布局挤占空间，导致卡片变形

**修复**:
- `TaskWizard.vue` 改为 `position: fixed`
- 设置 `top: 52px`, `right: 0`, `bottom: 0`, `z-index: 100`
- `WorkstationHome.vue` 调整布局，避免被挤压

**效果**: 向导覆盖而非挤压，主页面卡片保持宽度不变

---

### ✅ 3. 两个侧边栏同时显示

**问题**: 访问 `/envclaw` 时，Hermes 的 AppSidebar 和 envclaw 的侧边栏同时显示

**修复**:
- `App.vue` 新增 `isEnvclawRoute` 判断
- `showAppSidebar` 逻辑排除 envclaw 路由

**效果**: 访问 envclaw 时只显示一个侧边栏

---

### ✅ 4. 成功率假数据

**问题**: "今日成功率 98%" 是硬编码假值，但 UI 没有标识

**修复**:
- 将卡片改为 `pending` 状态
- 显示"—"和"待接入"徽章
- 保留 TODO 注释说明后续接入真实 API

**效果**: 用户不会误以为 98% 是真实数据
