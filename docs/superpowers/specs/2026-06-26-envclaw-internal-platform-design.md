# 数智大气内置化 + 提示词黑盒 + 设计语言升级

- **日期**：2026-06-26
- **状态**：设计确认，待实施
- **前置设计**：`2026-06-25-envclaw-differentiation-design.md`

---

## 1. 背景与动机

基于 `2026-06-25-envclaw-differentiation-design.md` 的设计，当前存在三个需要修正的问题：

1. **数智大气平台定位不清**：envclaw 本身就是数智大气的自动化值守层，用户登录即数智大气账号。但当前设计中数智大气作为普通平台出现在平台管理列表中，用户可以删除、编辑账号，与"内置平台"的定位矛盾。
2. **提示词黑盒不足**：当前 `PromptPreview.vue` 完整展示原始提示词文本，对业务人员来说是一堆难以理解的"咒语"，违背了"把提示词内化为平台配置"的设计初衷。
3. **设计语言缺乏温度**：当前黑白灰单色系 Naive UI 主题偏冷，需要增加温暖亲和的友好感与人性化的温柔细节，同时保持理性克制的专业感。

---

## 2. 数智大气作为内置平台

### 2.1 核心定位

数智大气是系统内置的特殊平台，在平台管理中可见但受保护——账号只读、功能不可删、平台不可删。新建任务向导的数据平台选择器正常包含数智大气，用户可勾选。

### 2.2 数据模型改动

```ts
export type PlatformType = 'mapairs' | 'national_station' | 'oa' | 'feishu' | 'dingtalk' | 'custom' | 'shuzhi'

interface Platform {
  // ... 现有字段
  builtIn?: boolean  // 是否内置平台（数智大气）
}

interface PlatformFunction {
  id: string
  name: string
  prompt: string
  builtIn?: boolean  // 是否内置功能，不可删除
}
```

### 2.3 平台管理页改动

**`PlatformsPage.vue`**：数智大气卡片正常展示，但交互受限。

**`PlatformCard.vue`**：
- 卡片右上角不显示"删除"按钮（`builtIn` 平台隐藏）
- 账号子行不显示"编辑""删除"按钮，仅展示账号名 + 状态 pill（只读）
- 账号子行末尾增加"系统绑定"标签（`NTag`，`type="default"`），提示用户账号由系统管理

**`PlatformConfigDrawer.vue`**：
- 功能列表中，`builtIn` 功能不渲染删除按钮
- 操作提示词、平台技能字段设为 `disabled`（`builtIn` 平台）
- 抽屉标题旁增加"内置平台"标签

### 2.4 账号绑定逻辑

- 数智大气账号 = 当前登录用户的数智大气会话
- 后端自动同步登录状态到平台账号的 `status` 字段
- 前端只读展示，不提供账号 CRUD 入口
- `AccountDrawer.vue` 对数智大气账号设为只读模式（所有字段 `disabled`）

### 2.5 新建任务向导

数据平台选择器正常包含数智大气，用户可勾选。选中后，数智大气的内置功能自动列出，用户可勾选功能（但不可删除功能本身）。

### 2.6 后端改动

- 初始化时自动创建数智大气平台实例（`type: 'shuzhi'`, `builtIn: true`）
- API 层对 `builtIn` 平台拒绝删除请求（返回 403）
- API 层对 `builtIn` 功能拒绝删除请求（返回 403）
- 账号 CRUD API 对 `builtIn` 平台返回只读数据

## 3. 提示词黑盒处理

### 3.1 现状问题

当前 `PromptPreview.vue` 完整展示组装后的原始提示词文本（角色基底 + 平台操作提示词 + 功能提示词），对业务人员来说是一堆难以理解的"咒语"。

### 3.2 PromptPreview 组件改造

移除原始提示词文本展示，改为**结构化摘要卡片**。

**摘要卡片展示内容**：

| 摘要项 | 展示内容 | 图标 | 标签色 |
|--------|----------|------|--------|
| 角色定位 | 一句话概括（如"数据播报助手"） | 机器人图标 | 蓝 `#6a9fd9` |
| 平台操作知识 | 平台名 + 一句话概括（如"数智大气：已加载登录流程与页面结构知识"） | 平台图标 | 琥珀 `#d4a574` |
| 已选功能 | 功能名 + 一句话说明（如"小时播报：每小时抓取并播报一次"） | 功能图标 | 绿 `#7cb97b` |
| 用户补充说明 | 用户自定义补充说明（如有） | 编辑图标 | 灰 `#a0a0a0` |

**摘要卡片样式**：
- 每个摘要项是一个小卡片/行，左侧带图标，右侧是标签 + 简短描述
- 卡片背景 `--envclaw-bg-tertiary`，圆角 8px，内边距 10px 12px
- 卡片之间间距 8px
- 整体视觉像"已为你准备好的执行清单"，而非"可编辑的提示词"

**补充说明保留**：用户的自定义补充说明仍然可编辑，但展示在摘要卡片下方，与组装部分视觉分离。

### 3.3 后端组装

- 前端只收集用户的配置选择（平台 ID 列表、功能 ID 列表、补充说明文本）
- 提交时，后端根据以下公式完成最终 prompt 组装：
  ```
  最终 prompt = 模板角色基底（模板自带）
              + 平台操作提示词（来自所选平台）
              + Σ 已选功能提示词（来自所选功能）
              + 用户补充说明（手填，可选）
  ```
- 前端不再持有完整的 prompt 文本，只在摘要中展示结构化信息
- `usePromptAssembly.ts` 改为后端组装，前端只负责摘要展示

### 3.4 影响范围

- `PromptPreview.vue`（完全重写为摘要卡片）
- `TemplateWizard.vue`（移除 prompt 文本相关逻辑，保留补充说明）
- `usePromptAssembly.ts`（改为后端组装）
- 后端 API（新增 prompt 组装逻辑）

## 4. 设计语言升级

### 4.1 方向

温暖与专业并重。在现有黑白灰 Naive UI 主题基础上，增加温度感和人性化细节，同时保持理性克制的专业感。

### 4.2 配色微调

**`theme.ts` 改动**：

| 变量 | 当前值 | 新值 | 说明 |
|------|--------|------|------|
| 暗色背景 | `#1a1a1a` | `#1c1b1a` | 略带暖意 |
| 暗色卡片 | `#2a2a2a` | `#2c2b29` | 略带暖意 |
| 亮色背景 | `#fafafa` | `#faf8f5` | 略带暖意 |
| 暖琥珀点缀 | — | `#c9a96e` | 新增，用于重要提示、完成状态、选中高亮 |
| 成功色 | `#66bb6a` | `#7cb97b` | 更柔和 |
| 警告色 | `#e0a458` | `#d4a574` | 更温暖 |
| 错误色 | `#d96a6a` | `#c97a7a` | 更柔和 |
| 信息色 | `#6a9fd9` | `#6a9fd9` | 保持 |

### 4.3 圆角与间距

| 元素 | 当前值 | 新值 | 说明 |
|------|--------|------|------|
| 卡片圆角 | 12px | 14px | 更柔和 |
| 输入框圆角 | 8px | 10px | 更柔和 |
| 卡片间距 | 12px | 16px | 增加呼吸感 |
| 字段间距 | 18px | 22px | 增加呼吸感 |

### 4.4 微交互

- **卡片 hover**：`box-shadow: 0 4px 20px rgba(0,0,0,0.08)` + `transform: translateY(-1px)`，过渡 0.2s ease
- **按钮 hover**：`transform: translateY(-1px)`，过渡 0.15s ease
- **功能卡片选中**：暖琥珀色 1.5px 边框 + 柔和背景高亮（`rgba(201,169,110,0.08)`）
- **表单输入 focus**：边框色过渡更柔和（0.2s ease）
- **平台卡片连接状态 pill**：成功状态增加柔和的呼吸灯效果（`box-shadow: 0 0 8px rgba(124,185,123,0.4)`，动画 2s ease-in-out infinite）

### 4.5 图标与按键样式

**图标**：
- 保持 Lucide 风格线性 SVG（`stroke-width` 1.6）
- 新增**填充图标**用于重要状态（如完成、成功），填充色使用暖琥珀 `#c9a96e`
- 图标与文字对齐方式：图标居中对齐，文字基线对齐
- 图标尺寸：小 14px、中 16px、大 20px

**按键样式**：
- 主按钮：背景渐变色（暗色 `linear-gradient(135deg, #e0e0e0 0%, #c9a96e 100%)`，亮色 `linear-gradient(135deg, #333333 0%, #555555 100%)`），文字色对比色（暗色按钮用 `#1c1b1a`，亮色按钮用 `#faf8f5`）
- 主按钮 hover：渐变偏移 + 微缩放
- 次要按钮：边框 1.5px（当前 1px），边框色 `--envclaw-border`，hover 时边框色变为暖琥珀
- 图标按钮：圆形背景（`border-radius: 50%`），hover 时背景色 `rgba(201,169,110,0.1)`
- 按钮内图标与文字间距 6px

### 4.6 细节人性化

- **空状态**：温暖插画/图标 + 友好文案（如"还没有配置平台，点击上方按钮添加第一个数据源"）
- **加载状态**：骨架屏替代旋转图标（`NSkeleton` 组件）
- **成功/错误提示**：柔和背景色 + 图标（`NMessage` 自定义样式）
- **表单 hint 文案**更友好（如"选好后，相关的功能会自动出现"）
- **平台卡片连接状态 pill**：成功状态增加柔和的呼吸灯效果（见 4.4）

### 4.7 字体与排版

- 保持 Inter + PingFang SC / Microsoft YaHei
- 标题字重 600 → 500（更柔和）
- 行高 1.5 → 1.6（更舒适）
- 段落间距增加 4px

### 4.8 影响范围

- `theme.ts`（全局主题变量）
- `PlatformCard.vue`（卡片 hover、状态 pill 呼吸灯）
- `PlatformsPage.vue`（空状态、间距）
- `TemplateWizard.vue`（字段间距、hint 文案）
- `PromptPreview.vue`（摘要卡片样式）
- `WorkstationHome.vue`（整体布局）
- 所有 envclaw 子组件（图标、按键样式）

## 5. 与前置设计的差异

| 维度 | 前置设计 | 本设计 |
|------|----------|--------|
| 数智大气定位 | 普通平台，可删除 | 内置平台，受保护 |
| 账号管理 | 多账号 CRUD | 数智大气账号只读，外部平台多账号 |
| 内置功能 | 平台配置中可删 | 标记 `builtIn`，不可删除 |
| 提示词展示 | 完整展示原始文本 | 结构化摘要卡片 |
| 提示词组装 | 前端组装 | 后端组装 |
| 设计语言 | 黑白灰单色系 | 暖灰 + 暖琥珀点缀 |
| 微交互 | 无 | 卡片 hover、按钮 hover、呼吸灯 |
| 图标 | Lucide 线性 | 线性 + 填充（重要状态） |
| 按键 | Naive UI 默认 | 渐变主按钮、圆角次要按钮 |
