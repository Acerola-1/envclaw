# 环保管家客户端差异化设计

- **日期**：2026-06-25
- **状态**：设计已确认，待评审
- **原型**：见同目录 `prototype-*.html`（工作台 / 向导 / 平台账号 / 智能问数）

---

## 1. 背景与问题

当前客户端基于 hermes-studio 二次开发（Electron，多端通用）。已做的优化包括：简化 Hermes 安装流程、更换核心组件下载源、内置 agent-browser CLI、内置 IPP-AIR 操作指南 skill。

**核心问题**：领导评价"差异化太少，本质是给 Hermes 套了个 UI"。具体痛点：

1. 入口仍是空白对话框，与官方 Hermes 无本质区别，预置功能只是多了几个可见入口。
2. 用户操作繁琐：创建无人值守任务仍要手写大段提示词，门槛高于直接用 Hermes。
3. 各平台（mapairs、国控站、OA、通讯工具）登录靠手动，每次重复操作。
4. 缺乏环保行业的垂直化体验，感觉是"通用 AI"而非"环保专业工具"。

**目标用户**：内部环保业务人员优先，逐步外推到外部客户（企业/政府）。这类用户熟悉环保业务但不熟悉 AI 工具。

**高频场景**：数据采集与报表、异常预警、数据分析与报告。

**一句话定位**：
> 打开就能用的环保智能管家——内置环保行业 know-how，靠交互式引导而非手写提示词完成自动化任务。

注意：目标不是"零提示词"（不现实），而是**把提示词内化为分步引导**——系统一步步问、用户一步步给关键信息，清晰明了。

---

## 2. 总体方案：双模式融合

顶栏提供两个**平级**入口，共享底层任务系统：

```
┌─────────────────────────────────────────────────────┐
│ 🌿 数智大气·环保管家   [工作台] [对话]      🔔 ⚙️ 👤 │
├─────────────────────────────────────────────────────┤
│  工作台模式：场景卡片 + 任务列表 + 平台账号 + 概览     │
│  对话模式：  完整 Hermes 对话能力                     │
└─────────────────────────────────────────────────────┘
```

**关键原则**：

- 两个模式共享同一个 jobs store 与任务数据。工作台创建的任务在对话中可见，反之亦然。
- 切换模式不丢失状态（Pinia 持久化）。
- 对话模式**完整保留** Hermes 能力，与工作台平级，不降级为二级入口（满足高级用户与不在预设场景内的需求）。

**为什么是 C 而不是 A/B**：

- 方案 A（仅在现有对话框架上加向导弹窗）差异化不足，仍是 Hermes 皮肤。
- 方案 B（纯工作台、去掉对话）差异化强但牺牲了自由对话，目标用户明确要求对话与工作台平级。
- 方案 C 在差异化与能力完整性之间取得平衡，且可渐进式迭代。

## 3. 模块设计

系统拆分为 5 个相互独立、通过明确接口协作的单元。每个单元可独立理解与测试。

### 3.1 工作台首页（Workstation Home）

**职责**：业务人员的默认落地页，把"能做什么"和"正在做什么"一屏呈现。

**组成**（自上而下）：

1. **欢迎概览** — 个性化问候 + 日期/单位 + 4 张统计卡（运行中任务、今日成功率、待处理预警、今日采集量）。
2. **业务场景卡片** — 4 张卡片：数据采集、异常预警、报告生成、智能问数。点击进入对应创建向导或问数页。
3. **活跃任务列表** — 运行中/暂停任务一览，含频率、推送渠道、上次运行时间、状态 pill、快捷操作（立即运行 / 更多）。
4. **平台账号条** — 各平台连接状态一目了然，点击进入平台管理。

**依赖**：jobs store（任务数据）、platform-accounts store（连接状态）、概览统计接口。

**新增 vs 复用**：新增 `WorkstationHome.vue` 及子组件；任务数据复用现有 `stores/hermes/jobs.ts`。

### 3.2 创建向导（Task Wizard）

**职责**：消除手写提示词的摩擦。提供两种创建路径，用户按场景自由选择。

**模式 A — 模板向导**（固定流程，适合明确的高频场景）：

- 分步表单（如数据采集为 4 步：选场景 → 基本配置 → 配置数据源 → 确认）。
- 每步提供**环保领域预设值**（河南省各地市、国标六项指标 PM2.5/PM10/O₃/SO₂/NO₂/CO、常用采集页面、常用频率与推送渠道），用户点选标签而非输入文本。
- 最后一步自动生成 prompt 与任务配置，用户确认即创建。

**模式 B — AI 引导**（自主规划，适合需求不在模板内或用户不确定如何配置时）：

- 类似 Claude Code 的交互：AI 一次只问一个问题，提供可点选的预设选项，也接受自由输入。
- AI 过程中注入环保领域知识（识别"平顶山"为城市、"PM2.5"为指标等）。
- 引导结束后给出**完整方案摘要卡**，用户可"调整"或"确认创建"。

**两种模式的边界**：模板向导覆盖固定结构化场景；AI 引导作为模板无法覆盖时的兜底与自由路径。向导面板顶部一键切换。

**形态**：向导为**右侧滑出面板**（非弹窗），左侧任务列表保持可见，便于对照。

**依赖**：场景模板定义（见 3.5）、platform-accounts store（列出可用账号）、jobs store（创建任务）、AI 引导后端接口。

**新增 vs 复用**：新增 `TaskWizard.vue`（含 TemplateWizard / AiGuide 两个子组件）；最终落库复用现有 job 创建 API。现有 `CreateGuardTaskModal.vue` 的逻辑可迁移/复用为模板向导的一种。

### 3.3 平台账号管理（Platform Accounts）

**职责**：集中管理各平台登录凭据，做到"配置一次，自动登录"。

**数据模型**：

```ts
interface PlatformAccount {
  id: string
  platform: 'mapairs' | 'national_station' | 'oa' | 'feishu' | 'dingtalk' | 'custom'
  name: string                 // 用户自定义，如"平顶山mapairs账号"
  credentials: {
    type: 'password' | 'api_key' | 'webhook' | 'cookie'
    username?: string          // 加密存储
    password?: string          // 加密存储
    apiKey?: string            // 加密存储
    webhookUrl?: string
    extra?: Record<string, string>
  }
  status: 'connected' | 'expired' | 'error' | 'pending'
  lastLogin?: string           // ISO 时间
  lastError?: string
  autoRefresh: boolean         // 会话过期是否自动用凭据重登
}
```

**安全设计**：

- 凭据以 AES-256 加密存储于本地 SQLite。
- 前端仅显示掩码（`••••••••`），不回传明文。
- 支持"连接测试"一键验证凭据有效性。
- 会话过期且 `autoRefresh` 为真时，用仍有效的凭据自动重新登录。

**多账号**：同一平台可配置多个账号（如多个地区的 mapairs 账号），向导创建任务时按平台列出可选账号。

**两层入口**：

1. 工作台底部账号条（展示 + 快捷入口）。
2. 独立"平台账号"页（完整管理：增删改、连接测试、多账号展开、编辑抽屉）。

**依赖**：本地加密存储（SQLite + AES-256）、各平台登录/测试的后端能力（经 agent-browser / Hermes 工具执行实际登录）。

**新增 vs 复用**：新增 `PlatformAccounts.vue`、`platform-accounts` store、加密存储后端模块、`platforms` API。与现有 `auth.ts`（应用自身用户体系）相互独立——平台凭据是"被自动化操作的外部站点凭据"，不要与应用登录混淆。

### 3.4 智能问数（Smart Query）

**职责**：自然语言查询环保数据，结构化展示结果。独立入口，**不走 Hermes 对话**。

**接入**：对接数值大气现有的智能体（而非当前 Hermes）。这是与官方 Hermes 的关键差异之一。

**交互**：

- 输入框 + 快速提问 chips（今日排名、趋势对比、达标情况、预警统计）。
- 左侧最近查询历史。
- 结果以**结构化卡片**渲染：KPI 卡、排名表（高亮关注城市）、折线图，而非纯文本。
- 支持"追问"：在上一次结果基础上继续深入；结果可"导出到报告"或"设为定时监控"。

**依赖**：数值大气智能体接口、图表渲染。

**新增 vs 复用**：新增 `SmartQuery.vue` 及结果渲染组件、`smart-query` API。

### 3.5 场景模板系统（Scenario Templates）

**职责**：把硬编码的值守机器人改为可配置的模板库，驱动场景卡片与模板向导。

**现状**：当前 6 个值守机器人硬编码在 `GuardPanel.vue` 中，文案未走 i18n。

**改进**：

- 模板定义为结构化数据（场景类型、图标、描述、向导步骤、各步骤预设选项、prompt 模板）。
- 内置环保领域模板（数据采集、异常预警、报告生成等），后续可扩展。
- 模板向导（3.2 模式 A）的步骤与预设值由模板定义驱动。

**依赖**：被工作台首页（场景卡片）与创建向导（模板）消费。

**新增 vs 复用**：新增模板定义模块；重构 `GuardPanel.vue` 的硬编码数据为模板驱动，并补齐 i18n。

## 4. 与官方 Hermes 的差异化总结

| 维度 | 官方 Hermes | 本产品 |
|------|------------|--------|
| 首页 | 空白对话框 | 业务场景看板 + 数据概览 |
| 创建任务 | 手写提示词 | 模板向导 / AI 引导（分步点选） |
| 平台登录 | 每次手动 | 配置一次，自动登录 + 多账号 |
| 行业知识 | 通用 AI | 内置环保领域预设（地市、国标指标、平台） |
| 智能问数 | 无 | 独立入口，接入数值大气专业智能体 |
| 值守模板 | 无 | 可配置场景模板库 |
| 用户取向 | 开发者向 | 业务人员向，点选为主 |

## 5. 设计语言

对齐项目现有 Naive UI 主题（见 `packages/client/src/styles/theme.ts`）：

- **配色**：黑白灰单色系（暗色 `--bg #1a1a1a`、卡片 `#2a2a2a`、主色 `#e0e0e0`）。状态色克制使用——绿 `#66bb6a`（正常/成功）、琥珀 `#e0a458`（警告/过期）、红 `#d96a6a`（错误/上升）、蓝 `#6a9fd9`（信息）。
- **字体**：Inter（中文回退 PingFang SC / Microsoft YaHei）。
- **圆角**：8px（小 6px、大 12px）。
- **图标**：Lucide 风格线性 SVG（`stroke-width` 1.6），**不使用 emoji**。项目当前用 Naive UI 组件，图标库待落地时统一选型（建议 `@vicons` 或本地 SVG 组件）。

## 6. 范围与边界（YAGNI）

**本期包含**：

- 双模式切换框架（工作台 / 对话）。
- 工作台首页（概览 + 场景卡片 + 任务列表 + 平台账号条）。
- 创建向导（模板向导 + AI 引导）。
- 平台账号管理（加密存储 + 多账号 + 连接测试 + 自动重登）。
- 智能问数页面（接入数值大气智能体）。
- 场景模板系统（重构现有硬编码值守模板）。

**本期不包含 / 后续规划**：

- 集成数值大气的用户系统与权限系统（独立 spec）。
- 智能问数后端智能体的实现（本期只对接接口，假定接口由数值大气提供）。
- 报告模板的可视化编辑器（本期报告生成走预设模板）。

**未决依赖**（需与后端/数值大气团队确认）：

- 数值大气智能体的接口契约（智能问数）。
- 各外部平台自动登录的技术可行性（经 agent-browser 维持会话）。
